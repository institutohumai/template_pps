# API Reference

## Introducción
Descripción de cómo interactuar con la API.

## Endpoints disponibles (si es una API web)

### 1. `GET /endpoint`
- **Descripción:** Obtiene información específica.
- **Parámetros:**
  - `id` (opcional): El ID del recurso.
- **Respuesta:**
  ```json
  {
    "id": 1,
    "name": "Ejemplo"
  }
  ```

### 2. `POST /endpoint`
- **Descripción:** Crear un nuevo recurso.

- **Respuesta:**
  ```json
  {
    "name": "Nuevo recurso"
  }
  ```
